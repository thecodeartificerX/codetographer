# Sanity Check — Implementation Plan Index

**Goal:** Add a diagnostic and auto-repair system for codetographer plugin state: a core `src/sanity.ts` module, session-start integration, a `/sanity` skill, and a CLI wrapper.

**Spec:** `docs/superpowers/specs/2026-03-25-sanity-check-design.md`

## Architecture Overview

```
src/sanity.ts (core module)
  ├── imports: map-generator.ts, atomic-write.ts, hooks/lib/recent-activity.ts
  ├── exports: runSanityCheck(options) → SanityReport
  └── 17 discrete checks, each with optional auto-fix

Integration:
  session-start.ts → calls runSanityCheck({ fix: true, skipExpensive: true, quiet: true })
  scripts/sanity.js → CLI wrapper, compiled from dist/sanity.js via copy-hooks.js
  skills/sanity/SKILL.md → full diagnostic + domain agent dispatch
```

**Key design constraint:** `updateRecentActivity()` is extracted from `stop.ts` into a new shared module `src/hooks/lib/recent-activity.ts`, imported by both `stop.ts` and `sanity.ts`. The domain parsing in `sanity.ts` reuses the same table-parsing approach as `domain-matcher.ts` and `mcp/server.js`.

## Tech Stack

- TypeScript (strict, ESM, NodeNext resolution)
- `node:test` + `node:assert/strict` for tests
- `tsx` for running tests against source
- Build: `tsc` → `dist/`, then `copy-hooks.js` patches imports

## Parallel Opportunities

- **Phase 2** tasks 1-3 (interfaces, individual check functions, domain parser) can be parallelized since they touch different parts of `src/sanity.ts` — but given it's a single file, sequential within the phase is safer.
- **Phase 3** tasks (copy-hooks.js patch + CLI wrapper) are independent and can run in parallel.
