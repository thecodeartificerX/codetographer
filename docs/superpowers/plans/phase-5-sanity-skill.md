## Gotchas
- The skill runs `scripts/sanity.js` via Bash tool — the script must already be generated by `npm run build:hooks` (Phase 3)
- The skill dispatches `domain-explorer` agents for stale/missing domains — reuse the exact agent spec from `agents/domain-explorer.md`

## Task 1: Create `/sanity` skill

**Description:** Create `skills/sanity/SKILL.md` as the skill definition for the `/sanity` command. This skill runs the full sanity check, displays results, dispatches agents for stale domains, and re-verifies.

**Acceptance criteria:**
- Frontmatter: `name: sanity`, description mentions sanity check/diagnostic/health check
- Trigger patterns: `/sanity`, "sanity check", "health check", "verify codetographer"
- The skill instructs the agent to:
  1. Run `node $CLAUDE_PLUGIN_ROOT/scripts/sanity.js --project-dir $CLAUDE_PROJECT_DIR --json` via Bash
  2. Parse the JSON output as a `SanityReport`
  3. Display the formatted report table (PASS/FIXED/WARN/FAIL per check)
  4. If `staleDomains` are found in any check result, collect the unique list
  5. For each stale/missing domain, dispatch a `domain-explorer` agent in parallel (Agent tool, model: sonnet)
  6. After all agents complete, re-run the sanity check via the same Bash command
  7. If map.md is stale in the re-check, run `node $CLAUDE_PLUGIN_ROOT/scripts/treesitter-map.js --root $CLAUDE_PROJECT_DIR --output docs/codetographer/map.md --budget 5000` to regenerate
  8. Display final status
- The skill should handle the case where the initial check returns `healthy` — just display the report and done, no agents needed
- The skill should handle the case where INDEX.md doesn't exist — suggest running `/codetographer` wizard instead

**Files:**
- `skills/sanity/SKILL.md` — CREATE

**Architecture notes:**

Report display format:
```
Codetographer Sanity Check
──────────────────────────
 PASS  docs/codetographer/ exists
 PASS  INDEX.md exists
 FIXED map.md regenerated (was stale)
 PASS  changes.md exists
 PASS  CLAUDE.md has ## Codetographer section
 FIXED hookify rule commit-before-stop.local.md restored
 PASS  hookify rule use-codetographer-docs.local.md exists
 PASS  Environment variables set
 PASS  Dependencies installed
 PASS  Hook scripts present
 WARN  2 stale domains: auth, api → dispatching explorers...

Status: fixed (11/12 passed, 2 fixed, 1 needs sync)
```

Domain explorer agent dispatch pattern (from existing wizard-flow.md):
- Use the Agent tool with `agents/domain-explorer.md` system prompt
- Pass the domain name, path patterns (from INDEX.md), and project dir
- Agent writes `docs/codetographer/domains/<domain>.md`
- Multiple agents can run in parallel

Post-agent re-check and map regen:
- Re-run sanity.js to confirm domain docs are fresh
- If map.md is still stale (expected due to post-tool-use hook writes), run treesitter-map.js
- Display final "Re-check after sync" table

**Validation:**
Manual: run `/sanity` in a test project and verify the report displays correctly.
