# Phase 3: Build Integration + CLI Wrapper

## Gotchas
- The `copy-hooks.js` Phase 5 pattern for `treesitter-map.js` uses: `from './<name>'` → `from '../hooks/dist/<name>'`. This is because `dist/sanity.js` has same-directory imports like `./map-generator.js`, and when the file moves to `scripts/`, those modules live in `hooks/dist/`.
- `scripts/sanity.js` is a **generated file** — it is produced by `copy-hooks.js` from `dist/sanity.js`. Do not hand-write it. The source of truth is `src/sanity.ts`.
- The CLI wrapper logic (arg parsing, JSON output, text formatting) lives in `src/sanity.ts` as an optional `main()` function gated by `import.meta.url` check, or as a separate `src/sanity-cli.ts`. The simpler approach is adding a `main()` at the bottom of `src/sanity.ts` that only runs when the file is the entry point.
- `process.argv` parsing should be minimal — no dependency on `commander` or `yargs`. Just parse `--flags` manually.

## Task 1: Add CLI entry point to `src/sanity.ts`

**Description:** Add a `main()` function to the bottom of `src/sanity.ts` that parses CLI arguments, runs the sanity check, and outputs results as JSON or formatted text. The main function should only execute when the file is run directly (not when imported as a module).

**Acceptance criteria:**
- `main()` is called at the bottom of the file, guarded by a check that this is the entry point (e.g., check if `process.argv[1]` ends with `sanity.js`)
- Parses flags: `--project-dir`, `--plugin-root`, `--plugin-data`, `--no-fix`, `--quiet`, `--skip-expensive`, `--json`
- Defaults: `projectDir` from `CLAUDE_PROJECT_DIR` or `process.cwd()`, `pluginRoot` from `CLAUDE_PLUGIN_ROOT`, `pluginData` from `CLAUDE_PLUGIN_DATA`
- `--json` outputs `JSON.stringify(report, null, 2)` to stdout
- Text output renders the formatted report table:
  ```
  Codetographer Sanity Check
  ──────────────────────────
   PASS  check description
   FIXED check description
   WARN  check description
   FAIL  check description

  Status: healthy (N/N passed)
  ```
- When `--quiet`, only non-`pass` checks are shown in text mode (but JSON always includes all)
- Exit code: 0 for healthy/fixed, 1 for needs_attention

**Files:**
- `src/sanity.ts` — MODIFY (add `main()` function and `formatReport()` helper)

**Architecture notes:**

Text output format:
- Status labels are right-padded: ` PASS `, ` FIXED`, ` WARN `, ` FAIL `
- Summary line: `Status: <status> (<counts>)`
- `staleDomains` in any check → append to the relevant check's message: `"2 stale domains: auth, api"`

Arg parsing pattern (no deps):
```ts
const args = process.argv.slice(2);
function getArg(name: string): string | undefined {
  const idx = args.indexOf(name);
  return idx >= 0 && idx + 1 < args.length ? args[idx + 1] : undefined;
}
const hasFlag = (name: string) => args.includes(name);
```

**Validation:** `npm run build` succeeds, `dist/sanity.js` contains the main function

## Task 2: Add `scripts/sanity.js` copy+patch step to `copy-hooks.js`

**Description:** Add a new block to `scripts/copy-hooks.js` that copies `dist/sanity.js` to `scripts/sanity.js` with import path patching, mirroring the existing `treesitter-map.js` pattern.

**Acceptance criteria:**
- New block added after the existing treesitter-map.js copy block (Phase 5 in copy-hooks.js)
- Reads `dist/sanity.js`, applies regex: `from './<name>'` → `from '../hooks/dist/<name>'`
- Also patches: `from './hooks/lib/<name>'` → `from '../hooks/dist/hooks/lib/<name>'` (for the `recent-activity.js` import)
- Writes result to `scripts/sanity.js`
- Logs `copied+patched: dist/sanity.js → scripts/sanity.js`
- Gracefully skips if `dist/sanity.js` does not exist (guards with `existsSync`)

**Files:**
- `scripts/copy-hooks.js` — MODIFY (add copy+patch block)

**Architecture notes:**

The import paths in `dist/sanity.js` will include:
- `from './map-generator.js'` → `from '../hooks/dist/map-generator.js'`
- `from './atomic-write.js'` → `from '../hooks/dist/atomic-write.js'`
- `from './hooks/lib/recent-activity.js'` → `from '../hooks/dist/hooks/lib/recent-activity.js'`

The existing `treesitter-map.js` regex (`/from '\.\/([^']+)'/g` → `"from '../hooks/dist/$1'"`) already handles this correctly for both cases since `./hooks/lib/recent-activity.js` matches `\.\/([^']+)` and becomes `../hooks/dist/hooks/lib/recent-activity.js`. So the **same single regex** works.

**Validation:**
```bash
npm run build:hooks
```
Verify `scripts/sanity.js` exists and contains patched imports (check that `../hooks/dist/` appears in the file).
